Scaling Vehicle Routing Problem

Project Member, IIT Madras

A popular graph problem involving the use of Minimum Spanning Tree(Kruskal's Algorithm) and Depth-First Traversal to find the most optimal route for distribution of ammenities to multiple nodes from a depot node with infinite capacity. This NP-HARD problem can be solved much faster by the use parallelization on GPUs which is explored in this paper.

Github

SYCLification of DFT-FE

Research Intern, IISc Bangalore

DFT-FE is a quantum mechanics concept used for modelling of materials at the lowest level of atoms. The existing DFT-FE code is a massively parallel one which uses more than 200k+ nodes to solve for millions of atoms. SYCL is a relatively new parallel programming language developed for Intel GPUs. This project studies the performance of SYCL compared to CUDA and HIP.

Github

Parallel Programming Projects


Parallelization of classic Lattice Boltzmann method

The Lattice Boltzmann method is a famous method used for solving many Computational fluid dynamics and transient heat conduction problems. This can be sped up by the use of parallelization. This paper details the use and efficiency of parallel programming in this topic. Results on OpenMP, MPI, hybrid OpenMP and MPI, CUDA(CuBLAS) and SYCL(MKL BLAS) have also been studied.

Github

Parallelization of Multigrid Method(V scheme)

The V-cycle scheme of the multigrid method is a technique for efficiently solving large-scale linear systems, often arising from PDEs in computational physics and engineering. This paper explores the implementation and optimization of parallel programming techniques applied to the V-cycle scheme using various frameworks, including OpenMP, MPI, OpenACC, CUDA, and SYCL.

Github

Parallel Cyclic Reduction of Tridiagonal systems

Parallelization of Cyclic Reduction Algorithm to solve Tridiagonal System of Linear Equation using GPU Programming Framework CUDA, OpenACC. The algorithm is divided into 3 steps, Forward elimination, Recursive Doubling and Back-substitution. The performance of the code was tested on Pade scheme in comparison with state of the art codes provided by NVIDIA.

Github
